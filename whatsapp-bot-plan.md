### 6.3 Vector Database + RAG Approach

**Knowledge Base Setup**

1. **Document Collection** (What goes into Vector DB):
   - All website content (services, pricing, FAQs)
   - Service descriptions (WhatsApp automation, event displays)
   - Case studies and examples
   - Common questions and answers
   - Company information and approach
   - Integration capabilities
   - Technical specifications

2. **Embedding Generation** (No API keys needed):
   ```python
   # Using local embedding model (sentence-transformers)
   from sentence_transformers import SentenceTransformer
   
   model = SentenceTransformer('all-MiniLM-L6-v2')  # Fast, lightweight
   # OR: 'paraphrase-multilingual-MiniLM-L12-v2' for ES/EN support
   
   # Generate embeddings for all knowledge base documents
   embeddings = model.encode(documents)
   ```

3. **Vector Database Storage**:
   ```python
   # ChromaDB (runs locally, no API needed)
   import chromadb
   
   client = chromadb.PersistentClient(path="./vector_db")
   collection = client.create_collection("company_knowledge")
   
   # Store documents with embeddings
   collection.add(
       documents=documents,
       embeddings=embeddings,
       metadatas=[{"source": "website", "type": "faq"}],
       ids=[f"doc_{i}" for i in range(len(documents))]
   )
   ```

**Query Processing Flow**

1. **User sends message** ‚Üí "How much does WhatsApp automation cost?"

2. **Embed user query** (using same local model):
   ```python
   query_embedding = model.encode([user_message])[0]
   ```

3. **Retrieve relevant documents** from Vector DB:
   ```python
   results = collection.query(
       query_embeddings=[query_embedding],
       n_results=3  # Top 3 most relevant documents
   )
   ```

4. **Generate response** (Two options):

   **Option A: Template-based (No LLM needed)**
   - Match to pre-written templates using similarity scores
   - Fast, predictable, no hallucinations
   - Best for FAQ-style questions
   
   **Option B: Local LLM (Optional, still no API)**
   ```python
   # Using Ollama with Llama 3 or Mistral (runs locally)
   import ollama
   
   context = "\n".join(results['documents'])
   prompt = f"""
   Based on this information:
   {context}
   
   Answer the user's question: {user_message}
   Keep it concise and friendly.
   """
   
   response = ollama.generate(model='llama3', prompt=prompt)
   ```

```python
}

def detect_intent(message):
    # First: Check keywords
    for intent, keywords in intents.items():
        if any(kw in message.lower() for kw in keywords):
            return intent
    
    # Second: Vector similarity to intent examples
    results = intent_collection.query(
        query_embeddings=[embed(message)],
        n_results=1
    )
    
    if results['distances'][0][0] < 0.5:  # High similarity
        return results['metadatas'][0]['intent']
    
    return "unknown"
```

```python
# Extract numbers (inquiry volume, budget)
```

```python
        documents=[c['text'] for c in new_chunks],
        embeddings=embeddings.tolist(),
        metadatas=[c['metadata'] for c in new_chunks],
        ids=[c['id'] for c in new_chunks]
    )
```

### F. Glossary

**Key Terms**

- **WhatsApp Business API**: Official API for businesses to automate WhatsApp
- **Conversation Flow**: Predetermined path of conversation
- **Intent**: User's goal or purpose (e.g., "schedule consultation")
- **Entity**: Specific data point extracted (e.g., "company name")
- **NLP**: Natural Language Processing - understanding human language
- **CRM**: Customer Relationship Management system
- **Lead Scoring**: Assigning value to leads based on criteria
- **CSAT**: Customer Satisfaction Score
- **Handoff**: Transferring conversation from bot to human
- **Session**: Time-limited conversation period (24 hours on WhatsApp)
- **Template Message**: Pre-approved message format (required for starting conversations)
- **Rich Media**: Images, videos, PDFs sent via chat
- **Webhook**: Endpoint that receives real-time updates
